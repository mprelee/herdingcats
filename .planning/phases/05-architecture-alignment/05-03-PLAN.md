Write README.md describing the HerdingCats architecture model. The README must cover all 8 core terms from ARCHITECTURE.md: Input, State, Behavior, Diff, Trace, Frame, Outcome, Engine. Do not introduce new terminology.

Structure:
1. **Title and tagline** — "HerdingCats" + one-line description: deterministic, turn-based game engine
2. **Overview** — 3-4 sentences: input-driven dispatch, static behavior set, deterministic ordering, atomic commits
3. **Core Concepts** — describe the 8 terms, each with 1-3 sentences:
   - Input: user-defined message submitted to the engine
   - State: the game state, provided at construction
   - Behavior: a rule evaluated during dispatch, returns diffs or stops
   - Diff: atomic state mutation, applied immediately during dispatch
   - Trace: metadata generated at the moment a diff is applied
   - Frame: canonical committed record {input, diffs, traces}
   - Outcome: result of dispatch/undo/redo — 7 variants (Committed, Undone, Redone, NoChange, InvalidInput, Disallowed, Aborted)
   - Engine: the coordinator — holds state, behaviors, history stacks
4. **How Dispatch Works** — numbered steps matching ARCHITECTURE.md:
   1. Caller submits Input with Reversibility declaration
   2. Engine creates CoW working state (no clone until first diff)
   3. Behaviors evaluated in deterministic (order_key, name) order
   4. Each behavior emits diffs (Continue) or halts (Stop with NonCommittedOutcome)
   5. Diffs applied immediately to working state, traces emitted at application time
   6. If diffs produced: state committed atomically, Frame created, Outcome::Committed returned
   7. If no diffs: Outcome::NoChange returned, no state change
5. **Undo/Redo** — snapshot-based, no reverse-diff trait required; irreversible transitions clear history
6. **Quick Start** — minimal code example showing EngineSpec impl, Behavior impl, Apply impl, Engine::new, dispatch
7. **Zero Dependencies** — note this is a zero-dependency library (proptest is dev-only)

Keep the README under 200 lines. Use the 8 core terms consistently. Do not add badges, CI links, or contribution guidelines.
