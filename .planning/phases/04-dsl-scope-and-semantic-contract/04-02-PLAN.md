<interfaces>
pub trait Operation<S>: Clone {
    fn apply(&self, state: &mut S);
    fn undo(&self, state: &mut S);
    fn hash_bytes(&self) -> Vec<u8>;
}

pub trait Rule<S, O, E, P>
where
    S: Clone,
    O: Operation<S>,
    P: Copy + Ord,
{
    fn id(&self) -> &'static str;
    fn priority(&self) -> P;
    fn before(&self, _state: &S, _event: &mut E, _tx: &mut Transaction<O>) {}
    fn after(&self, _state: &S, _event: &E, _tx: &mut Transaction<O>) {}
}
</interfaces>

Write the pre-codegen validation contract and the consumer binding/config contract.

`docs/dsl/validation.md` must specify semantic failures that happen before Rust emission, including:
- duplicate or unstable rule ids
- unsupported lifetime values or forms
- event references outside the approved matching surface
- guard expressions outside the approved binding surface
- effects that imply direct state mutation
- lossy effects that cannot prove reversibility
- any construct that implies mutating `after()` behavior

`docs/dsl/bindings.md` must specify:
- what consumer state/event paths or helper names the DSL may reference

The binding contract must be specific enough that Phase 5 can implement it without changing the runtime engine API.
