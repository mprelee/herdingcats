**src/rule.rs — /// on Rule trait and all methods (DOC-01, DOC-03, DOC-04):**

Rule trait doc: 3-5 sentences — the most important doc block alongside Operation. Must cover:
1. What a Rule is: an observer/modifier of in-flight transactions
2. The two-phase hook contract: before() sees the event before apply, after() sees state after apply
3. How rules compose: sorted by priority, all enabled rules run in order for before(), reverse order for after()
4. How rules are parameterized: S=state, O=operation, E=event, P=priority type

A reader who only reads the Operation and Rule trait docs should understand the whole engine model.

Method docs:
- `id()`: must return a stable &'static str — used as the rule's identity key in the engine
- `priority()`: determines execution order; lower values run first in before(), last in after()
- `before()`: called before ops are applied — rules can push ops to tx or set tx.cancelled
- `after()`: called after ops are applied — rules can push additional ops for side effects

Rule trait `/// # Examples` block: show a minimal complete Rule impl (unit struct, both before and after implemented). Must be fully runnable with minimal inline types.
