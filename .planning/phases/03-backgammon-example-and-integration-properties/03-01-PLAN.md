Board encoding (BACK-02, locked):
```
board[0..=23]: points (positive = White count, negative = Black count)
board[24]:     White bar (positive count)
board[25]:     Black bar (negative count — Black checkers on bar stored as negative i8)
white_home:    u8 — White checkers borne off
black_home:    u8 — Black checkers borne off
```

Standard starting position (White moves toward higher indices):
```
White: +2 on point 23, +5 on point 12, +3 on point 7, +5 on point 5
Black: mirror (negative values at mirrored indices)
board[0] = -2, board[11] = +5, board[16] = -3, board[18] = -5
board[23] = +2, board[12] = +5, board[7] = +3, board[5] = +5
(and the negatives for Black's side)
```

Conservation invariant (BACK-05 helper, needed here):
```rust
fn checker_count(state: &BgState) -> u32 {
    let board_sum: u32 = state.board.iter().map(|x| x.unsigned_abs() as u32).sum();
    board_sum + state.white_home as u32 + state.black_home as u32
}
// Must equal 30 for any valid state
```

MoveOp undo logic (BACK-04, four variants):
```
1. Place on empty: apply board[to]+=sign / undo board[to]-=sign
2. Hit blot (captured=true): apply board[to]=sign + bar[opponent]+=1 / undo reverse
3. Re-enter from bar: apply board[bar]-=sign + board[to]+=sign / undo reverse
4. Bear off: apply board[from]-=sign + *home_counter+=1 / undo reverse
   NOTE: BearOffOp is a SEPARATE variant from MoveOp — never writes board[26]
In ALL cases: apply sets dice_used[die_index]=true, undo sets dice_used[die_index]=false
```
