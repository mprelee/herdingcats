### Integration Finding: EXAM-01 / InvalidInput (design note, not blocker)

`Outcome::InvalidInput` is defined in the type system and has an exhaustive match arm in `tictactoe.rs` (line 294–298) with an inline comment explaining it is structurally unreachable from `dispatch()` in the MVP engine. `BehaviorResult::Stop(info)` maps exclusively to `Outcome::Aborted(info)` — there is no code path in `Engine::dispatch` that constructs `InvalidInput`. The variant exists for future extension or for callers to handle should the engine evolve. This is a known, documented design choice logged during Phase 4 planning.
