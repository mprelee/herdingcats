Public API (use herdingcats::*):
```rust
// EngineSpec: bundle all type params
pub trait EngineSpec: Sized {
    type State: Clone + Debug + Default;
    type Input: Clone + Debug;
    type Diff: Clone + Debug + Apply<Self>;
    type Trace;
    type NonCommittedInfo;
    type OrderKey: Ord;
}

// Behavior: zero-size structs
pub trait Behavior<E: EngineSpec> {
    fn name(&self) -> &'static str;
    fn order_key(&self) -> E::OrderKey;
    fn evaluate(&self, input: &E::Input, state: &E::State) -> BehaviorResult<E::Diff, E::NonCommittedInfo>;
}

// Apply: impl on Diff type
pub trait Apply<E: EngineSpec> {
    fn apply(&self, state: &mut E::State) -> Vec<E::Trace>;
}

// Engine
pub struct Engine<E: EngineSpec>;
impl<E: EngineSpec> Engine<E> {
    pub fn new(state: E::State, behaviors: Vec<Box<dyn Behavior<E>>>) -> Self;
    pub fn state(&self) -> &E::State;
    pub fn undo_depth(&self) -> usize;
    pub fn redo_depth(&self) -> usize;
    pub fn dispatch(&mut self, input: E::Input, reversibility: Reversibility)
        -> Result<Outcome<Frame<E>, E::NonCommittedInfo>, EngineError>;
    pub fn undo(&mut self) -> Result<Outcome<Frame<E>, HistoryDisallowed>, EngineError>;
    pub fn redo(&mut self) -> Result<Outcome<Frame<E>, HistoryDisallowed>, EngineError>;
}

// Outcome — NOT #[non_exhaustive], all 7 variants are stable
pub enum Outcome<F, N> {
    Committed(F),
    Undone(F),
    Redone(F),
    NoChange,
    InvalidInput(N),
    Disallowed(N),
    Aborted(N),
}

pub enum BehaviorResult<D, O> { Continue(Vec<D>), Stop(O) }
pub enum Reversibility { Reversible, Irreversible }
pub enum HistoryDisallowed { NothingToUndo, NothingToRedo }

// Frame — manual Clone/PartialEq (not derive — E does not implement Clone/PartialEq)
pub struct Frame<E: EngineSpec> {
    pub input: E::Input,
    pub diffs: Vec<E::Diff>,
    pub traces: Vec<E::Trace>,
    pub reversibility: Reversibility,
}
```

CRITICAL PITFALL — InvalidInput from dispatch:
BehaviorResult::Stop(info) always maps to Outcome::Aborted(info) in the dispatch loop.
Outcome::InvalidInput is NEVER produced by dispatch in the MVP engine.
Outcome::Disallowed IS produced by undo()/redo() on empty stacks.
Solution: ValidateCell returns Stop("out of bounds") → Aborted; ValidateTurn returns Stop("wrong player") → Aborted.
The output formatter has an exhaustive match with an InvalidInput arm that prints [InvalidInput] — it is structurally present but currently unreachable from dispatch.

```rust
use herdingcats::{Apply, Behavior, BehaviorResult, Engine, EngineError, EngineSpec,
                  Frame, HistoryDisallowed, Outcome, Reversibility};

struct TicTacToeSpec;

#[derive(Debug, Clone, PartialEq, Default)]
enum Player { #[default] X, O }

impl Player {
    fn other(&self) -> Player { match self { Player::X => Player::O, Player::O => Player::X } }
}

#[derive(Debug, Clone, Default)]
struct TicTacToeState {
    board: [[Option<Player>; 3]; 3],
    current_player: Player,
    game_over: bool,
}

#[derive(Debug, Clone)]
enum TicTacToeInput {
    Place { row: usize, col: usize },
}

#[derive(Debug, Clone)]
enum TicTacToeDiff {
    PlaceMarker { row: usize, col: usize, player: Player },
    SwitchPlayer,
    SetGameOver,
}
```

**Apply impl** — PlaceMarker sets board cell and returns trace "placed X at (r,c)"; SwitchPlayer toggles current_player and returns trace "switched to P"; SetGameOver sets game_over=true and returns trace "game over".

**EngineSpec impl** — TicTacToeSpec with State=TicTacToeState, Input=TicTacToeInput, Diff=TicTacToeDiff, Trace=String, NonCommittedInfo=String, OrderKey=u32.

2. `ValidateCell` (order_key=1): Check row/col bounds (0..3); if out of bounds → Stop("out of bounds"). Check if cell is occupied; if occupied → Stop("cell already occupied").
3. `ValidateTurn` (order_key=2): Check if input player matches current_player. For scripted demo, embed player in the input or derive from state. Since the input is just Place{row,col}, the "player" is always the current player — ValidateTurn becomes a no-op. Instead: wire the scripted sequence so wrong-turn is triggered by dispatching twice for same player.
4. `PlaceMarker` (order_key=3): Emits PlaceMarker diff + SwitchPlayer diff.
5. `CheckWin` (order_key=4): After placement, check rows/cols/diagonals. If 3-in-a-row: emit SetGameOver diff and Stop("X wins") to produce Aborted... wait, Stop produces Aborted not Committed. Use Continue([SetGameOver]) instead to commit the win.

```rust
fn print_dispatch(label: &str, result: &Result<Outcome<Frame<TicTacToeSpec>, String>, EngineError>) {
    match result {
        Ok(Outcome::Committed(frame)) =>
            println!("[dispatch] {} => Committed (diffs: {}, traces: {})",
                     label, frame.diffs.len(), frame.traces.len()),
        Ok(Outcome::NoChange) =>
            println!("[dispatch] {} => NoChange", label),
        Ok(Outcome::Aborted(reason)) =>
            println!("[dispatch] {} => Aborted({})", label, reason),
        Ok(Outcome::InvalidInput(reason)) =>

        Ok(Outcome::Undone(_)) | Ok(Outcome::Redone(_)) => unreachable!("dispatch never returns Undone/Redone"),
        Err(e) => println!("[dispatch] {} => EngineError({:?})", label, e),
    }
```
