**src/behavior.rs:**
1. Remove the `Behavior` trait entirely (lines 93-119) and all related rustdoc referencing dyn-safety, `Box<dyn Behavior>`, trait methods.
2. Keep `BehaviorResult<D, N>` enum unchanged (it is correct).
3. Add `BehaviorDef<E: EngineSpec>` as a plain struct:
   ```rust
   pub struct BehaviorDef<E: EngineSpec> {
       pub name: &'static str,
       pub order_key: E::OrderKey,
       pub evaluate: fn(&E::Input, &E::State) -> BehaviorResult<E::Diff, E::NonCommittedInfo>,
   }
   ```
4. Implement `Debug` manually for `BehaviorDef<E>` (fn pointers don't derive Debug nicely — print name and order_key, show evaluate as `fn(...)` placeholder).
5. Update module-level rustdoc: remove all references to the Behavior trait, dyn-safety, `Box<dyn Behavior>`. Emphasize: static compile-time behavior set, deterministic ordering, behaviors return diffs only, engine applies diffs centrally. Mention `BehaviorDef<E>` is a plain struct with fn pointers — no trait objects needed.
6. Add a rustdoc example on `BehaviorDef` showing construction with a closure-like fn pointer.
7. Update the `compile_fail` doctest: instead of testing that `Behavior::evaluate` prevents mutation, show that a `BehaviorDef` evaluate fn pointer receives `&E::State` (shared borrow) preventing mutation. The test structure is similar but references `BehaviorDef` not `Behavior`.
8. Update tests module: remove `PassthroughBehavior`, `StoppingBehavior` structs and their `impl Behavior` blocks. Replace with tests that construct `BehaviorDef<TestSpec>` values directly. Keep the `TestSpec` and `Apply<TestSpec> for u8` impl. Test: constructing a BehaviorDef, accessing its fields, calling its evaluate fn pointer. Remove `behavior_is_dyn_safe` test entirely. Update remaining tests to use `BehaviorDef` construction.

**src/engine.rs:**
1. Change import: `use crate::behavior::BehaviorDef;` instead of `use crate::behavior::{Behavior, BehaviorResult};`. Also import `BehaviorResult` from behavior.
2. Change `Engine<E>` field: `behaviors: Vec<BehaviorDef<E>>` (not `Vec<Box<dyn Behavior<E>>>`).
3. Update `Engine::new()` signature: accept `Vec<BehaviorDef<E>>` instead of `Vec<Box<dyn Behavior<E>>>`.
4. Update sort in `new()`: sort by `(b.order_key.clone(), b.name)` using struct field access instead of method calls. Since `order_key` is `E::OrderKey: Ord + Clone`, clone for comparison.
5. Update `dispatch()` loop: change `behavior.evaluate(&input, &*working)` to `(behavior.evaluate)(&input, &*working)`. The parentheses around `behavior.evaluate` are required because Rust parses `behavior.evaluate(...)` as a method call, but `evaluate` is a field containing a fn pointer — use `(behavior.evaluate)(...)`.
6. Update ALL doctests on `Engine` struct and methods: replace `impl Behavior<MySpec> for AppendBehavior { ... }` with `BehaviorDef { name: "append", order_key: 0, evaluate: |input, _state| BehaviorResult::Continue(vec![AppendDiff(*input)]) }` — but note fn pointers require `fn` not closures. Use a named function: `fn append_eval(input: &u8, _state: &Vec<u8>) -> BehaviorResult<AppendDiff, String> { BehaviorResult::Continue(vec![AppendDiff(*input)]) }` then `BehaviorDef { name: "append", order_key: 0, evaluate: append_eval }`.
7. Update ALL `#[cfg(test)]` tests: replace `Box::new(SomeBehavior)` patterns with `BehaviorDef { name: ..., order_key: ..., evaluate: some_fn }`. For each test behavior (CountBehavior, StopBehavior, etc.), convert to a standalone `fn` that captures the same logic, then construct `BehaviorDef` with that fn. Remove all `struct SomeBehavior` and `impl Behavior<TestSpec> for SomeBehavior` blocks in tests.
8. Update all test helper structs: any struct that existed solely to implement `Behavior` should become a plain fn.

**src/lib.rs:**
1. Change `pub use crate::behavior::{Behavior, BehaviorResult};` to `pub use crate::behavior::{BehaviorDef, BehaviorResult};`.
2. Update module-level rustdoc: replace "Implement [`Behavior`]" with "Define [`BehaviorDef`] entries". Replace "Construct an `Engine` (Phase 2)" with "Construct an `Engine`".
