Replace the current single-line `//! herdingcats — a deterministic, undoable game-event engine.` tagline with a full 3-4 paragraph `//!` block. Keep the tagline as the opening sentence.

Structure:

**Paragraph 1 — State machine model (Mealy/Moore framing):**
Describe the engine as a Mealy machine: state is the automaton's internal memory; events (inputs of type `I`) drive transitions; mutations (outputs of type `M`) transform state; behaviors implement the transition function. Each event produces a new state AND a set of mutations that encode the transition. Undo reverses the mutation sequence.

**Paragraph 2 — Four concepts with one-line roles:**
- `Engine<S,M,I,P>` — the automaton: holds state, runs behaviors, commits actions
- `Behavior<S,M,I,P>` — the transition function: translates events into mutations via `before`/`after` hooks
- `Mutation<S>` — the output: atomic, invertible state change; `apply` advances, `undo` reverses
- `Action<M>` — the transition batch: collects mutations for a single dispatch; reversible iff all mutations are

**Paragraph 3 — ASCII dispatch pipeline:**
```
Dispatch pipeline
─────────────────
Event (I)
  │
  ├─► [Behavior.before()]  ← inject/cancel mutations
  │     (ascending priority)
  │
  ├─► [Mutation.apply()]   ← advance state S → S'
  │
  └─► [Behavior.after()]   ← react to new state S'
        (descending priority)

Undo path (reversible actions only)
────────────────────────────────────
[Mutation.undo()]    ← restore S' → S
[Behavior.on_undo()] ← reverse behavior-internal state
```
