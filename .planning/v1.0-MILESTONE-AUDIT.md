## Tech Debt

### Non-Critical Items

**Phase 1:**
- `src/transaction.rs` doctest contains `// A Transaction<i32> where the op type is i32 (placeholder type here)` — informational comment in a doctest, not a stub. All 15 doctests pass. No action needed.

**All Phases:**
- `tx.irreversible` field naming inversion: `true` means "push to undo stack" (i.e., the transaction IS reversible), `false` means "skip stack" (non-reversible). The name reads opposite to the behavioral effect. This is a pre-existing design choice, documented in `src/transaction.rs` doc comments and consistently understood across all three phases. No change is needed for v1.0 but worth tracking for a potential `tx.record_undo_frame` rename in v2.

### Total: 2 minor items across all phases — no blockers
